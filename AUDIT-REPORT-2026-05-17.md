# Design Audit Report — 2026-05-17

Generated by parallel multi-skill audit (Impeccable v3.1.1 + ui-ux-pro-max v2.5.0 + design-for-ai) across student, admin, public/PDF, jobs-ops, and deterministic-CLI tracks. Scope: `apps/admissions/src/`, `apps/jobs-ops/src/`, and `apps/admissions/public/fonts/pdf/` (PDF system).

## Executive summary

- **Total surfaces audited:** student (12 files), admin (~25 files), public/landing/PDF/jobs-ops (~42 files), plus full deterministic CLI scan of both apps.
- **Findings:** **1 P0**, **17 P1**, **14 P2** (deduplicated across tracks).
- **The dominant issue is systemic**, not localised: raw `slate-*` Tailwind classes in 329 instances across 45 files, bypassing the token system. Fixing this is mechanical but high-volume.
- **3 cross-cutting fixes** would resolve roughly 70% of findings: (1) `slate-*` → semantic-token sweep, (2) `--color-scrim` token introduction + `bg-black/N` replacement, (3) deletion of `.gradient-text*` legacy CSS classes + their single consumer.
- **No high-severity safety regressions** were found. Auto-save, virtualised admin queue, bulk-action confirmation gates, and skeleton-not-spinner discipline are all in place.

## Methodology

Three skills, four parallel auditors, one synthesis pass:

| Skill | What it caught |
|-------|----------------|
| **Impeccable** | Anti-pattern register-mode violations (gradient-text, AI tells, decorative motion, side-tab cards). Caught the gradient-text wordmark and the legacy CSS classes. |
| **ui-ux-pro-max** | UX guidelines compliance (priority 1–10): accessibility, touch, performance, layout, typography, motion, forms, navigation, data-viz. Recommended palette comparison vs current tokens. |
| **design-for-ai** | Kadavy CHECKER mode: typography, proportions, composition, visual hierarchy, color theory. Caught the two-CTAs-equal-weight issue on the landing hero. |

Cross-reference: every finding mapped against `PRODUCT.md` anti-references and `DESIGN.md` token catalogue. CLI gate (`impeccable detect` 29 deterministic rules) provides a reproducible per-file report.

## P0 — Must fix before next release

### 1. `gradient-text-primary` on the institutional wordmark

- **Surface:** Public, plus every authenticated page that renders `SharedFooter`
- **Where:** `apps/admissions/src/components/layout/SharedFooter.tsx:41`
- **Lens:** Impeccable (CLI rule `gradient-text`), PRODUCT.md anti-references (gradient text in headings)
- **Why P0:** The wordmark identifies the institution. Gradient text on "MIHAS-KATC" reads as a tech-startup move, not a medical school. PRODUCT.md says brand voice is *calm and institutional*. This single line of code violates the most foundational rule — and the underlying CSS class is in the canonical `design-tokens.css` file, marked "for backward compatibility".
- **Fix:**
  1. Replace `className="text-xl sm:text-2xl font-bold gradient-text-primary"` with `className="text-xl sm:text-2xl font-bold text-primary"` (or `text-foreground` if the footer is on a colored background).
  2. Delete `.gradient-text`, `.gradient-text-primary`, and `.gradient-accent` from `design-tokens.css:435-447`.
  3. `bun test` to confirm no other consumers.
- **Effort:** **S (15 min)**

#### 3. Introduce `--color-scrim` token, replace 18 `bg-black` modal scrims

- **Surface:** all (modal/dialog primitives + product-specific dialogs)
- **Where:** `Dialog.tsx:19`, `alert-dialog.tsx:18`, `ConfirmDialog.tsx:43,81`, `DataPopulationConfirmation.tsx:38`, `Modal.tsx`, `FormFeedback.tsx:113`, `Toast.tsx:328`, `BulkActionsBar.tsx:100,105`, `ApplicationDetailModal.tsx:310,821`, `NotificationBell.tsx:122`, `MobileBottomNav.tsx:130,170`, `BaseNavigation.tsx`, `DesktopSidebar.tsx`, `KeyboardShortcutsHelp.tsx:37`, `DraftManager.tsx:83`, `SubmissionSuccess.tsx:92,97`, plus 1 in jobs-ops `UnsavedChangesDialog.tsx:19`
- **Lens:** Impeccable CLI (`pure-black-white`)
- **Why P1:** Pure black scrims feel harsh and slightly off. Brand-tinted near-black unifies modal language. Bulk-action and approval modals especially benefit — they should sell gravity, not surprise.
- **Fix:**
  1. Add to `design-tokens.css`:
     ```css
     --color-scrim: 12 14 24;  /* RGB triplet, oklch(12% 0.01 250) */
     ```
  2. Add to Tailwind config:
     ```js
     scrim: 'rgb(var(--color-scrim-rgb) / <alpha-value>)',
     ```
  3. Sweep `bg-black/50` → `bg-scrim/50`, `bg-black/80` → `bg-scrim/80` across all 18 instances.
  4. Visual diff in dev — confirm scrim still looks dark enough (oklch(12%) is very close to pure black).
- **Effort:** **M (2 h)**

#### 4. Remove decorative animations: `animate-bounce`, bounce-easing, layout-transition

- **Surface:** all
- **Where:** `styles/smooth-animations.css:5,14`, `styles/mobile-enhancements.css:587 (bounce)`, `styles/mobile-enhancements.css:254 (transition: width,height)`, `components/student/NotificationBell.tsx:104`
- **Lens:** Impeccable, ui-ux-pro-max §7 motion (`spring-physics`, `transform-performance`, `excessive-motion`)
- **Why P1:** Bounce easing is dated, layout-thrashing, and competes with information density. The notification bell needs no bounce — count change + color change is enough signal.
- **Fix:**
  1. Delete `animate-bounce` from `NotificationBell.tsx:104`. If a transition is wanted, use `transition-transform duration-200` + `scale-105` only at the moment of count update.
  2. Delete `.animate-bounce` keyframes and `.bounce` classes from `smooth-animations.css` and `mobile-enhancements.css`.
  3. Replace `transition: width, height` in `mobile-enhancements.css:254` with `grid-template-rows: 0fr → 1fr` for height collapse, or `transform: scale()` for size emphasis.
- **Effort:** **M (3 h)** including visual regression check

### Student surface

#### 5. Wizard `staggerChild` — verify reduced-motion compliance

- **Where:** `pages/student/Dashboard.tsx` imports `staggerChild` from `lib/animations`
- **Lens:** ui-ux-pro-max §1 `reduced-motion`
- **Why P1:** Global CSS `prefers-reduced-motion` clamps CSS animation, but if `staggerChild` uses `framer-motion` directly, it must call `useReducedMotion()` to opt out.
- **Fix:** Read `lib/animations.ts`. If `staggerChild` doesn't already short-circuit on reduced motion, wrap.
- **Effort:** **S (30 min)**

#### 6. Wizard payment step — confirm icon-only buttons have aria-labels

## Strengths to preserve

1. **Canonical primitive composition is consistently followed.** `PageShell` + `SectionCard` + `ErrorDisplay` + `EmptyState` is the pattern across every student page. Don't break this in any refactor.
2. **Auto-save infrastructure is robust.** `draftManager`, `applicationSession`, `useApplicationDirty`, and `SaveStatusIndicator`'s 6-state machine cover the full failure space.
3. **Skeleton-not-spinner discipline.** Multiple view-specific skeletons + `AppShellSkeleton`, `GuardInlineSkeleton`. Loading is layout-aware.
4. **Virtualised admin queue.** `VirtualizedApplicationsGrid` exists explicitly. Admin queue scales.
5. **Filter persistence via URL params.** `useApplicationFilters` + `useSearchParams`. Refresh-survivable.
6. **Bulk action enforcement.** SHA-256 confirmation token, 25-app batch limit, all-or-nothing transaction.
7. **Idempotency keys on mutations.** Replay-safe.
8. **Speculative prefetch wired correctly.** `onDashboardMount` triggers wizard chunk preload during dead time.
9. **Error reporting end-to-end.** `ErrorBoundary`, `reportError`, `logApiError`, plus `Seo`, plus GlitchTip integration.
10. **WCAG AA-documented tokens.** Every contrast ratio inline in `design-tokens.css`. Many tokens exceed AA into AAA.
11. **Reduced-motion globally enforced** in `index.css`.
12. **Touch-first tailwind utilities** (`min-h-touch`, `min-w-touch`).
13. **Two separate PDF systems used correctly.** Branded student documents (Playfair Display + Source Sans 3 in `@react-pdf/renderer`) for letters and receipts; jspdf for large admin tabular exports.
14. **Drift-guard tests in CI** for status enums, role hierarchy, error codes, payment-status mapping. Already prevents new drift; just doesn't yet block design drift (CLI gate would close that gap).

## What this audit did NOT cover

- **Visual regression** — no Playwright/Percy snapshot pass against production design.
- **PDF rendered output** — `@react-pdf/renderer` produces a binary PDF, not HTML. The visual fidelity of `ApplicationSlip`, `PaymentReceipt`, `AcceptanceLetter` against their design intent was not verified — the auditors only read the React-PDF source.
- **i18n / RTL** — language fallbacks, character-set support, RTL layout were not audited.
- **Print stylesheets** — `styles/print.css` exists; not audited for actual print output (needs paper test).
- **Email templates** — Python email components in `backend/apps/common/email/` not audited (not in frontend scope).
- **Bundle size** — only flagged as a finding; not measured. Run `vite build --mode production && du -sh dist/assets/*.js` to quantify.
- **Reduced-motion compliance for JS animations** (Framer Motion, etc.) — only the global CSS override was confirmed; per-import library compliance flagged for follow-up.
- **Light/dark mode** — dark mode is disabled at the HTML level; this audit assumed light-only.

   ```json

   ```
   Add a GitHub Actions step that runs this on PRs. Exit non-zero when any rule fires. This prevents new drift while the systemic backlog is worked off.

   - **P0 #1** (gradient-text wordmark) — 15 min, immediate visible improvement.
   - **P1 #4** (decorative animation removal) — 3 h, single PR.

   - Wizard payment a11y audit (#6)
   - Dashboard bundle audit (#7)
   - BulkActionsBar visual escalation (#8)
