# Historical Reports

Point-in-time audit, production-readiness, and hardening reports. These are
**historical records** of work already completed — kept for the audit trail, not
living documentation. For current state see:

- `README.md` (repo root) — platform overview.
- `docs/canonical-truth-map.md` — single source of truth per domain concept.
- `docs/multi-tenant-beanola-progress.md` — multi-tenant rollout progress + evidence.
- `.kiro/steering/` — authoritative steering (product, tech, structure, infrastructure).
- `docs/runbooks/` — operational runbooks.

The canonical April-2026 repository audit (`AUDIT-REPORT-2026-04-24.md`) and its
file inventory (`all-files.txt`) remain at the repo root because they are
referenced directly by the steering files.
