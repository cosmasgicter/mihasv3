# MIHAS API Schema Baseline

Authoritative OpenAPI 3.0.3 baseline for the Django REST API under `/api/v1/`.
Used by the CI `api-quality` job to detect regressions and breaking changes.

## Files

| File | Purpose | Regenerated by |
|---|---|---|
| `openapi.v1.baseline.yaml` | Canonical schema, locked as of each release cut | `make schema-baseline` |
| `lint_baseline.json` | Output of `.kiro/skills/api-design-reviewer/scripts/api_linter.py --format json` | `make api-quality-baseline` |
| `REMEDIATION_TARGETS.md` | Remediation baseline vs target metrics | manual + `backend/tests/unit/test_api_quality_script.py` |
| `AUDIT_CLOSE_OUT.md` | Before/after scorecard after each remediation sprint | `make api-quality-closeout` |

## Rules

- **Do not manually edit** `openapi.v1.baseline.yaml` or `lint_baseline.json`.
  These files are the "ground truth" for CI regression detection.
- Regenerate only when rolling a new baseline after a sprint completes:
  ```bash
  make schema-baseline
  ```
- Any PR that changes the baseline must include the rationale in the PR description
  and pass `breaking_change_detector.py` against the previous baseline.

## Regenerating the baseline

Prerequisites: a Python venv with the backend requirements installed (see `backend/requirements.txt`).

```bash
cd backend && \
  DJANGO_SETTINGS_MODULE=config.settings.dev \
  SECRET_KEY=x JWT_SIGNING_KEY=x \
  DATABASE_URL='sqlite:///tmp/audit.db' \
  REDIS_URL='redis://localhost:6379/0' \
  ALLOWED_HOSTS='*' \
  python manage.py spectacular --file schema/openapi.v1.baseline.yaml
```
