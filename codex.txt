SECTION 4 — DOCUMENT GENERATION & R2 MIGRATION (MANDATORY)
-----------------------------------------------------------
Target: 100% functional document generation and retrieval via Cloudflare R2.

Opus must perform:
1) Runtime path audit
- Identify every code path generating/storing/downloading documents.
- Confirm each path routes through current API contract and R2 backend.

2) Legacy URL inventory
- Query for old Supabase/legacy document URLs in application_documents and related fields.
- Export mapping: document_id -> old_url -> new_r2_key/new_url.

3) Backfill migration job
- For each legacy object:
  a) fetch original file,
  b) upload to R2 with deterministic key,
  c) verify checksum/size,
  d) update DB reference in transaction,
  e) log outcome.

4) Verification
- Random sample test downloads after migration.
- Confirm student dashboard buttons work for migrated and newly generated docs.

5) Safety
- Keep migration idempotent (re-runnable).
- Keep rollback metadata (old URL snapshot table/file).


SECTION 5 — LOOPHOLE CLOSURE CHECKLIST

- Inconsistent auth/session/profile object shapes across pages.
- Any direct legacy storage/database client usage in active runtime paths.
